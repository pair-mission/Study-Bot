# 프로젝트 개요

해당 프로젝트는 스터디의 모임 일정을 관리하고, 참여자들에게 리마인드 및 출석을 안내하는 콘솔 기반 애플리케이션입니다.
사용자는 콘솔 메뉴를 통해 모임을 등록, 수정, 삭제하고, 참가자를 추가하거나 출석을 기록하며, 일정 주기 및 알림 설정을 할 수 있습니다.

# 프로젝트 목표

콘솔의 기본적인 MVP 기능(Level 1)에서 시작하여,
'모임 일정 관리 → 출석 관리 → 자동 알림 기능’(Level 3)까지 점진적으로 확장합니다.

이 과정을 통해 다음을 학습하는 것을 목표로 합니다.

- 객체지향 도메인 설계와 계층별 책임 분리
- 설계를 리팩토링하며 구조를 단계적으로 개선하는 경험
- 파일 입출력을 통한 데이터 영속화
- 외부 라이브러리 없이 직접 기능을 구현하면서 라이브러리/프레임워크의 필요성을 체감하는 경험
- 페어 프로그래밍을 통한 설계 논의, 코드 리뷰, 의사소통 등 협업의 경험

### **Level 1 - 기본 기능 (MVP)**

스터디 일정과 멤버를 관리하고, 콘솔에서 일정 리마인드 메시지를 확인할 수 있다.

| **기능**     | **설명**                                |
|------------|---------------------------------------|
| 모임 등록      | 스터디 모임의 날짜, 시간, 주제, 장소를 받아서 모임을 등록한다. |
| 모임 삭제 및 수정 | 등록된 스터디 모임을 삭제하거나 변경한다.               |
| 모임 전체 조회   | 등록된 스터디 모임의 전체 목록을 조회한다.              |
| 멤버 조회      | 스터디에 참여 중인 모든 멤버를 조회한다.               |
| 멤버 등록      | 새로운 멤버를 추가한다.                         |
| 참여자 등록     | 멤버는 닉네임을 기반으로 참여하고 싶은 모임에 참여자로 등록한다.  |
| 참여자 조회     | 모임에 참여하는 전체 멤버를 조회한다.                 |
| 내 모임 조회    | 멤버는 자신이 참여하는 모임의 일정을 모아서 조회한다.        |
| 일정 리마인드    | 내일 열리는 스터디가 있으면 콘솔에 리마인드 메시지를 보여준다.   |

### **Level 2 – 출석 및 히스토리 관리**

모임의 참석 여부를 기록하고 히스토리를 관리할 수 있다.

| **기능**   | **설명**                                       |
|----------|----------------------------------------------|
| 출석 체크    | 스터디 시간에 맞춰 스터디 참여자들은 콘솔에서 참석 또는 불참 여부를 입력한다. |
| 출석 기록 저장 | 모임별 출석 기록을 저장한다. (파일로 내보내기)                  |
| 출석 기록 조회 | 지난 회차별 출석 기록을 조회한다.                          |
| 다음 모임 조회 | 멤버가 참여하는 모임 중, 가장 가까운 다음 모임의 정보를 보여준다.       |
| 알림 설정    | 스터디 시작 전 “일 단위”로 알림 조건 설정이 가능하다.             |

### **Level 3 – 외부 서비스 연동 및 데이터 자동 수집**

외부 서비스를 연동하여 자동화 기능을 추가한다.

| **기능**     | **설명**                                           |
|------------|--------------------------------------------------|
| 일정 자동 감지   | 시스템 시간 기반으로 알림을 보낼 대상을 자동으로 탐지하고 리마인드 이벤트를 생성한다. |
| 메시징 API 연동 | Discord/Slack으로 리마인드 메시지를 자동으로 전송한다.             |
| 데이터 영속화    | 파일로 출석 이력을 외부로 export한다. (.json, .txt 등)         |
| 스터디 리포트 생성 | 주간 출석률/활동 통계 리포트를 자동으로 만든다.                      |

# 프로젝트 개발 방식

## 릴레이 페어 프로그래밍

### 진행 방식

페어 프로그래밍 방식을 기반으로, 각 기능을 구현할 때마다 driver와 navigator를 번갈아가며 구현하는 방식으로 진행했습니다.
한 사람이 하나의 기능을 구현하면 다음 사람이 pull 받아서 다음 기능을 이어가는 식으로 릴레이를 진행합니다.

### 제한 조건

생성형 AI를 사용하지 않는다.

## 기능 목록

### 모임 관리

- 모임 등록
    - [x] 스터디 모임의 날짜, 시작 및 종료 시간, 주제, 장소를 입력 받아 모임을 등록한다.
    - 예외 상황
        - [x] 모임의 날짜, 시작 및 종료 시간, 주제, 장소 중 정보가 하나라도 없으면 예외가 발생한다.
        - [x] 모임의 날짜가 현재 기준으로 과거인 경우 예외가 발생한다.
        - [x] 날짜의 형식을 "YYYY-MM-dd"로 입력하지 않으면 예외가 발생한다.
        - [x] 모임의 종료 시간이 시작 시간보다 빠른 경우 예외가 발생한다.
        - [x] 시간의 형식을 "HH:mm"으로 입력하지 않으면 예외가 발생한다.
        - [x] 모임 시간의 시간(Hour)은 24시 표기법이어야 한다.
        - [x] 모임 시간의 분(Minute)은 30분 단위여야 한다.
- 모임 수정
    - [x] 모임의 ID와 새로운 정보(주제 또는 장소)를 입력 받아 모임의 정보를 수정한다.
        - 예외 상황
            - [ ] 존재하지 않는 모임이면 예외가 발생한다.
- 모임 삭제
    - [x] 모임의 ID를 입력 받아 모임을 삭제한다.
    - 예외 상황
        - [ ] 입력값을 숫자로 변환할 수 없으면 예외가 발생한다.
        - [ ] 존재하지 않는 모임이면 예외가 발생한다.
- 모임 전체 조회
    - [x] 등록된 모임 전체를 조회한다.

### 멤버 관리

- 멤버 등록
    - [x] 닉네임을 입력 받아 새로운 멤버를 등록한다.
    - 예외 상황
        - [x] 이미 등록한 멤버인 경우 예외가 발생한다.
- 멤버 조회
    - [x] 현재 스터디에 등록된 멤버 전체를 보여준다.
- 로그인
    - [x] 닉네임을 입력 받아 스터디 멤버인지 확인한다.
    - 예외 상황
        - [x] 해당 닉네임을 가진 스터디 멤버가 없는 경우 예외가 발생한다.

### 참여자 관리

- 참여자 등록
    - [x] 참여할 모임의 번호를 입력한다.
    - 예외 상황
        - [x] 입력한 값이 번호가 아니면 예외가 발생한다. // controller 테스트해야함
        - [ ] 입력한 값이 음수이면 예외가 발생한다. // 구현 안되어있음
        - [ ] 이미 참여 중인 경우 예외가 발생한다. // 구현 안되어있음
        - [x] 참여할 모임이 없는 경우 예외가 발생한다.
- 참여자 조회
    - [x] 조회할 모임의 번호를 입력한다.
    - 예외 상황
        - [x] 입력한 값이 번호가 아니면 예외가 발생한다. // controller 테스트해야함
        - [ ] 해당 모임이 존재하지 않을 경우 예외가 발생한다.

### 출석 기능

- 출석 등록
    - [x] 출석할 모임의 번호를 입력한다.
    - 예외 상황
        - [ ] 이미 출석한 모임이면 예외가 발생한다.
        - [ ] 모임 시작 시간 10분 전 후가 아니면 예외가 발생한다.
        - [ ] 참여 중이 아닌 모임에 출석 시 예외가 발생한다.
        - [ ] 해당 모임이 존재하지 않을 경우 예외가 발생한다.

- 출석 조회
    - [x] 메뉴를 선택해 모든 모임의 출석을 조회한다.

### 일정 리마인드

- 일정 변경
    - [x] 원하는 주기를 일 단위로 입력하여 주기를 변경한다.
      예외 상황
        - [ ] 입력 값이 숫자가 아니면 예외가 발생한다.
        - [ ] 입력 값이 음수인 경우 예외가 발생한다.

### 전체 흐름 제어

- 메뉴 입력
    - [x] 메뉴 번호를 입력 받아 해당 기능을 수행한다. 종료를 누를 때까지 메뉴 입력을 반복한다.
    - [x] 메뉴를 잘못 입력한 경우 예외 메시지를 출력하고 다시 입력받는다.
    - 예외 상황
        - [x] 입력값이 숫자 형태가 아니면 예외가 발생한다.
        - [x] 입력값이 존재하지 않는 메뉴 번호이면 예외가 발생한다.
        - [ ] 메인 메뉴 실행 중 에러가 발생하면 에러 메시지 출력 후 다시 메뉴를 입력 받는다.

### 공통 입력값 검증

- [x] 입력값이 빈 문자열, 공백, 또는 null이면 예외가 발생한다.
- [ ] 입력값이 유효한 숫자가 아니면 예외가 발생한다. (0 이상의 자연수만 허용)
